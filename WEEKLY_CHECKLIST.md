# SEO Content Growth - Weekly Checklist

Checklist settimanale per stare in track con la content strategy.

---

## 📋 WEEKLY ROUTINE (5-7 hours/week)

### MONDAY: Planning & Outline
- [ ] Review last week's analytics (Google Analytics 4)
- [ ] Outline this week's article (2000-3000 words)
- [ ] Plan social media content (3 LinkedIn posts, 5 tweets)
- [ ] Check keyword rankings (using Ahrefs/Ubersuggest)

**Time**: 1-2 hours

---

### TUESDAY-THURSDAY: Writing & Publishing
- [ ] Write article draft (main content)
- [ ] Edit & proofread
- [ ] Add code examples
- [ ] Add internal links to other posts
- [ ] Review SEO:
  - [ ] Title includes main keyword
  - [ ] Meta description (120-160 chars)
  - [ ] Headers properly structured (H1→H2→H3)
  - [ ] Images have alt text
- [ ] **Publish** to blog platform
- [ ] Update sitemap.xml (if manual)

**Time**: 2-3 hours

---

### FRIDAY: Social & Community
- [ ] Share article on LinkedIn (with custom caption)
- [ ] Share article on Twitter (create thread version)
- [ ] Share in relevant Reddit communities (r/angular, r/typescript)
- [ ] Answer 5 Stack Overflow questions (link blog if relevant)
- [ ] Comment on 3 Dev.to/Medium articles related to your topic
- [ ] Reply to comments/questions in your blog

**Time**: 1-1.5 hours

---

### SATURDAY: Backlink Building (alternating)
**Week 1-2**: Guest post pitching
- [ ] Identify 2 platforms for guest post
- [ ] Write + send pitch to editor
- [ ] Follow up on pending pitches

**Week 3-4**: Community building
- [ ] Answer 10+ Stack Overflow questions
- [ ] Engage on Reddit (ask questions, help others)
- [ ] Retweet + engage on Twitter
- [ ] Build relationships with dev influencers

**Time**: 1-1.5 hours

---

## 📅 MONTHLY REVIEW (1.5 hours)

### Analytics Review
- [ ] Google Search Console:
  - [ ] New keyword impressions?
  - [ ] Which posts are getting clicks?
  - [ ] Any indexing errors?
- [ ] Google Analytics:
  - [ ] Total traffic this month
  - [ ] Top posts by views
  - [ ] Average session duration
  - [ ] Bounce rate
- [ ] Ahrefs/Ubersuggest:
  - [ ] Keyword ranking changes
  - [ ] New backlinks acquired?
  - [ ] Competitor analysis

### Content Review
- [ ] Update 1-2 old blog posts with new info
- [ ] Add internal links from recent posts to older ones
- [ ] Fix any broken links

### Strategy Adjustment
- [ ] What topics drove the most traffic?
- [ ] Update next month's content calendar based on performance
- [ ] Adjust guest post strategy if needed

## 🔥 EMERGENCY: Slow Progress?

If after 3 months you're not seeing traction, check:

- [ ] Are your articles actually ranking for keywords?
  - Check Google Search Console → Performance
  - If not appearing in top 100: Keyword is too competitive or article isn't good enough
  - Solution: Pick easier keywords (long-tail) or improve article quality

- [ ] Are people staying on your blog?
  - Check Google Analytics → Scroll depth, Time on page
  - If bounce rate > 70%: Content isn't matching search intent
  - Solution: Improve intro, add more code examples, better formatting

- [ ] Are you getting shares?
  - Check LinkedIn/Twitter activity
  - If low: Content might be boring or not targeting audience pain points
  - Solution: Ask on Twitter "What Angular topic frustrates you most?"

- [ ] Is your site technically broken?
  - Check Google Search Console → Coverage (indexing errors)
  - Common issues: robots.txt blocking, duplicate content, slow loading
  - Solution: Fix indexing issues first

**If still stuck**: Post on r/webdev asking for feedback on articles.

## 🎁 BONUS: Content Ideas When You're Stuck

**Quick wins (500-1000 word articles):**
- FAQ post: "5 Angular Questions Beginners Ask (Answered)"
- Comparison: "Angular vs React: When to Use Each"
- Tool review: "5 Extensions I Use Daily in VSCode"
- Mistakes: "3 Common TypeScript Mistakes & How to Fix Them"
- List post: "10 RxJS Operators Explained"

**Deep dives (2000+ words):**
- Case study: How you built a featured project
- Tutorial: Step-by-step guide to complex feature
- Architecture post: How to structure large apps
- Benchmark: Performance comparison of approaches

**Evergreen content (never expires):**
- Best practices guides
- Language/framework fundamentals
- Tool comparisons
- Pattern explanations
